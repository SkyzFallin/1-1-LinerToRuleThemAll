# Security Policy

## Supported Versions

This project currently supports security fixes on the latest commit on the default branch.

## Reporting a Vulnerability

If you discover a vulnerability in this repository:

1. **Do not** open a public GitHub issue.
2. Open a private security advisory in GitHub (`Security` tab) if available.
3. If security advisories are unavailable, contact the maintainer directly via GitHub profile contact methods.
4. Include clear reproduction steps, impact, and any proof-of-concept details.

We aim to acknowledge reports within **72 hours** and provide a status update as triage progresses.

## Scope Notes

This tool includes offensive-security one-liners and is intended for authorized testing only. Security reports should focus on:

- Vulnerabilities in the Python application itself.
- Supply-chain or dependency risks in repository automation.
- Leaks of sensitive project or maintainer information.
